# All-In-One Securityの推奨設定と負荷

All-In-One Security（AIOS）は、ログイン保護からファイアウォールまで、多くの機能を1つにまとめたセキュリティプラグインです。機能が多いぶん、すべてを一度に有効化すると、サーバーに負荷がかかり、正常な操作が妨げられることがあります。この記事では、AIOSの機能を段階的に有効化する順番と、機能の重複や負荷を見ながら設定する方法を整理します。有効にすべき機能を、自分のサイトに合わせて選べるようにします。

## AIOSはどんなプラグインか

All-In-One Security（AIOS）は、多機能な総合型のセキュリティプラグインです。名前のとおり、セキュリティに関する多くの機能を1つにまとめています。バックアッププラグインのUpdraftPlusと同じ開発チームが手がけています。

AIOSが備える機能は、ログイン保護、ファイアウォール、ファイルの変更検知、スパム対策など、多岐にわたります。これらを1つのプラグインで扱えるのが特徴です。ただし、機能が多いことは、利点であると同時に、設定の複雑さや負荷という課題も生みます。

AIOSの主な機能を整理すると、次のようになります。それぞれ、守る対象が異なります。

| 機能分野 | 主な内容 |
|---|---|
| ログイン保護 | ログインロックダウン、ログインページの変更、CAPTCHA |
| ユーザー管理 | admin名の検出、ユーザー名の露出防止、表示名の確認 |
| ファイアウォール | PHPベースのファイアウォールルール |
| ファイル保護 | ファイルの変更検知、権限の確認 |
| スパム対策 | スパムコメントの防止、IPの遮断 |

これだけの機能があるため、AIOSは1つで多くの役割をカバーできます。しかし、すべてを一度に有効化すると、問題が生じることがあります。以降、段階的に設定する方法を説明します。

## 一度にすべてを有効化しない

AIOSを使う際に最も重要な原則は、すべての機能を一度に有効化しないことです。段階的に、1つずつ有効化し、そのつどサイトの動作を確認してください。

なぜ段階的にするのか。理由は2つあります。1つは、機能によっては正常な操作を妨げることがあるためです。設定を強くしすぎると、自分が管理画面に入れなくなったり、サイトの一部が動かなくなったりします。もう1つは、複数の機能を同時に有効化すると、問題が起きたときに原因を特定できないためです。

AIOSには、設定の目安として、基本・中級・上級といったセキュリティレベルの考え方があります。まず影響の小さい設定から始め、サイトが正常に動くことを確認しながら、段階的に強めていくのが安全です。

段階的な有効化の進め方は、次のとおりです。この順で進めると、問題が起きても切り分けられます。

1. まず、影響の小さい機能から有効化する
2. サイトの表示と、管理画面へのログインを確認する
3. 問題がなければ、次の機能を有効化する
4. 各段階で、動作を確認しながら進める

この進め方を守れば、万一問題が起きても、直前に有効化した機能が原因だと分かります。一度にすべてを有効化すると、この切り分けができません。

## 段階1：ログイン保護から始める

最初に有効化すべきは、ログイン保護の機能です。効果が大きく、比較的安全に導入できるためです。ただし、ログインに関わる設定は、自分を締め出すリスクもあるため、慎重に進めます。

AIOSのログイン保護には、いくつかの機能があります。まず有効にすべきものから、順に説明します。

- **ログインロックダウン**：連続したログイン失敗で、IPアドレスを遮断します。総当たり攻撃を防ぐ、基本的な機能です
- **admin名の検出**：既定の管理者名 admin が使われている場合に、それを検出します。変更を促してくれます
- **ユーザー名の露出防止**：author permalinkなどからユーザー名が取得されるのを防ぎます

ログインロックダウンは、まず有効にすべき機能です。ただし、遮断の条件を厳しくしすぎると、自分の打ち間違いでも遮断されます。適度な回数と時間に設定してください。

ログインページの変更機能もありますが、これは有効化すると自分のログインURLも変わります。有効にする場合は、変更後のURLを必ず記録してください。忘れると、ログイン画面にたどり着けなくなります。

## 段階2：ユーザーとファイルの保護を加える

ログイン保護が安定したら、次にユーザー管理とファイル保護の機能を加えます。これらは、比較的安全に有効化できる機能です。

この段階で有効にする主な機能は、次のとおりです。サイトの動作への影響が小さいものが中心です。

- **表示名とログイン名の確認**：表示名がログイン名と同じ場合に、それを検出します。同じだと、ユーザー名が露出するため、変更を促します
- **ファイルの変更検知**：ファイルが、通常の操作以外で変更された場合に通知します。改ざんの検知に役立ちます
- **ファイル権限の確認**：ファイルやフォルダの権限が、安全でない設定になっていないかを確認します

ファイルの変更検知は、改ざんに気づくために有用な機能です。ただし、通知が多すぎると、重要なものを見落とします。通知の設定を調整し、本当に注意すべき変更だけを通知するようにしてください。

これらの機能は、サイトの動作を妨げることが少ないため、比較的安全に有効化できます。有効にしたら、通知が適切に届くかを確認してください。

## admin名とユーザー名露出への対応

AIOSには、ログインの安全性を高める補助的な機能があります。とくに、既定の管理者名の検出と、ユーザー名の露出防止は、総当たり攻撃への備えとして有用です。

これらの機能は、攻撃者に手がかりを与えないための対策です。ユーザー名が知られると、攻撃者はパスワードだけを試せばよくなり、総当たり攻撃の成功率が上がります。AIOSは、この手がかりを減らす機能を持っています。

- **admin名の検出と変更の案内**：既定の管理者名 admin が使われている場合、それを検出し、変更を促します
- **表示名とログイン名の一致の検出**：表示名がログイン名と同じ場合、ユーザー名が露出するため、変更を促します
- **ユーザー名の露出防止**：author permalinkなどから、ユーザー名が取得されるのを防ぎます

これらは、いずれも設定の負荷が小さく、サイトの動作への影響もほとんどありません。有効にしておくことをおすすめします。

とくに、admin名が残っている場合は、変更を検討してください。admin は攻撃者が最初に試すユーザー名であり、これが使われていると、総当たり攻撃の標的になりやすくなります。AIOSの案内に従って、別の名前の管理者アカウントに移行してください。

## 段階3：ファイアウォールを慎重に有効化する

最後に、ファイアウォールを有効化します。ファイアウォールは効果が大きい反面、正常な操作を誤って遮断することがあるため、最も慎重に扱う必要があります。

AIOSのファイアウォールは、PHPベースのルールで、悪意のあるリクエストを遮断します。段階的に強度を選べるようになっており、弱いルールから順に適用できます。いきなり最も強いルールを適用すると、正常な操作まで遮断されることがあります。

ファイアウォールを有効化する際の注意点は、次のとおりです。強度を段階的に上げてください。

- **弱いルールから始める**：まず基本的なルールを適用し、サイトが正常に動くことを確認します
- **段階的に強める**：問題がなければ、より強いルールを適用します
- **サーバーのWAFとの重複に注意**：サーバー側でWAFが有効な場合、二重になって誤検知が増えます
- **除外設定を活用する**：正常な操作が遮断された場合、除外設定で調整します

とくに、サーバー側のWAFとの重複に注意してください。レンタルサーバーがWAFを提供している場合、AIOSのファイアウォールと重複します。両方を有効にすると、処理が重くなり、誤検知も増えます。どちらか一方に絞ることを検討してください。

ファイアウォールで正常な操作が遮断された場合は、その操作を除外する設定を追加します。強度を上げすぎず、サイトが正常に動く範囲で運用してください。

## 機能の重複を確認する

AIOSは多機能なため、他のプラグインやサーバーの機能と重複しやすい特徴があります。重複を放置すると、競合や無駄が生じます。導入前と導入後に、重複を確認してください。

重複しやすい機能と、その対処は次のとおりです。同じ役割の機能が複数あると、問題の原因になります。

| 重複しやすい機能 | 重複の相手 | 対処 |
|---|---|---|
| ファイアウォール | サーバーのWAF | どちらか一方に絞る |
| ログイン保護 | 他のログイン保護プラグイン | AIOSに一本化するか、他方を無効化 |
| ユーザー名の露出防止 | 他のセキュリティ設定 | 二重に設定しない |

とくに注意すべきは、他の防御系プラグインとの重複です。AIOSは総合型のため、ログイン保護やファイアウォールを備えています。同じ機能を持つ別のプラグインを併用すると、競合して管理画面に入れなくなることがあります。

AIOSを導入する場合は、他の防御系プラグインを無効化してください。防御系のプラグインは、1つに絞るのが原則です。AIOSに一本化すれば、多くの機能を1つでまかなえます。

サーバー側のWAFとの重複も確認してください。レンタルサーバーがWAFを提供している場合、AIOSのファイアウォールと重複します。両方を有効にせず、どちらを使うかを決めてください。

## サーバー負荷を意識する

AIOSは多機能なため、有効にする機能が増えるほど、サーバーへの負荷も増えます。とくに、常時動作する機能や、スキャンを行う機能は、負荷に影響します。表示速度への影響を意識してください。

負荷に影響しやすい機能と、その対策は次のとおりです。すべてを有効にするのではなく、必要なものを選んでください。

- **マルウェアスキャン**：ファイルをスキャンする機能は、実行時に負荷がかかります。頻度を調整します
- **ファイアウォール**：すべてのリクエストを検査するため、常時負荷がかかります
- **ファイルの変更検知**：定期的にファイルを確認するため、実行時に負荷がかかります

負荷が気になる場合は、スキャンの頻度を下げる、常時動作する機能を絞る、といった調整をしてください。AIOSの機能をすべて有効にする必要はありません。自分のサイトに必要な機能だけを選ぶことが、負荷と安全性のバランスを取る鍵です。

表示速度が明らかに落ちた場合は、有効にしている機能を見直してください。機能の多さは、運用できて初めて意味を持ちます。使わない機能まで有効にして負荷をかけるより、必要な機能に絞るほうが、結果として良い運用になります。

## 二要素認証の扱い

AIOSには、二要素認証の機能もあります。管理者アカウントには、この機能を有効にする価値があります。ただし、導入には注意点があります。

二要素認証を有効にすると、ログイン時に、パスワードに加えて認証コードが必要になります。パスワードが漏れても侵入を防げる、効果の高い機能です。AIOSでは、特定のユーザー権限に対して、二要素認証を求める設定ができます。

二要素認証を導入する際の注意点は、次のとおりです。締め出しを防ぐ準備が必要です。

- **復旧手段を確認する**：認証アプリが使えなくなった場合の復旧方法を、事前に確認します
- **管理者から導入する**：まず管理者アカウントで試し、問題がないことを確認します
- **サーバー接続情報を用意する**：締め出された場合に備え、FTPでプラグインを無効化できるようにします

二要素認証は効果が大きい反面、端末を紛失すると自分が締め出されます。導入前に、復旧手段を確認してください。締め出された場合は、FTPでAIOSのフォルダ名を変更すれば、無効化できます。

## WordPress側の設定も確認する

AIOSでサイトを守ると同時に、WordPressの基本的な設定も確認してください。AIOSの機能と、基本設定の両面から、状態を把握することが重要です。

## よくある質問

**Q. AIOSの機能は、すべて有効にすべきですか。**
A. いいえ。必要な機能だけを、段階的に有効にしてください。すべてを一度に有効化すると、負荷や誤動作の原因になります。

**Q. ファイアウォールを有効にしたら、管理画面が使えなくなりました。**
A. ファイアウォールが正常な操作を遮断した可能性があります。除外設定を追加するか、強度を下げてください。サーバー側のWAFとの重複も確認します。

**Q. AIOSは重いと聞きましたが、本当ですか。**
A. 多機能なため、有効にする機能が増えると負荷も増えます。必要な機能に絞れば、負荷は抑えられます。

**Q. 他のセキュリティプラグインと併用できますか。**
A. 防御系のプラグインは、1つに絞ってください。AIOSと他の防御系を併用すると、競合します。

## まとめ

All-In-One Security（AIOS）は、多機能な総合型のセキュリティプラグインです。ログイン保護、ファイアウォール、ファイル保護など、多くの機能を1つで扱えます。最も重要なのは、すべての機能を一度に有効化しないことです。ログイン保護から始め、ユーザーとファイルの保護を加え、最後にファイアウォールを慎重に有効化する。この順で、動作を確認しながら段階的に進めてください。機能が多いぶん、有効にするほどサーバー負荷も増えます。必要な機能に絞り、負荷と安全性のバランスを取ることが、AIOSを使いこなす鍵になります。
