# WordPressセキュリティプラグイン無料版8選

WordPressセキュリティプラグインの無料版でも、ログイン保護・WAF・マルウェアスキャン・二要素認証は利用できます。有料版との差は機能の有無ではなく、脅威情報の更新頻度とサポートの範囲にあります。この違いを知らずに無料版を使うと、守れていない部分を守れているつもりになります。8製品を役割ごとに整理し、無料で足りる条件を示します。

## 無料版でどこまで守れるのか

WordPressセキュリティプラグインの無料版で使えるのは、既知の攻撃パターンの遮断、ログイン保護、マルウェアスキャン、設定の診断です。実際に発生する侵害の多くは更新の放置と弱い認証情報から起こるため、この範囲でも大半の攻撃は防げます。

一方、無料版には共通した限界があります。それは機能が削られていることではなく、**新しい脅威に対応する速度**です。

代表例がWordfenceです。公式の説明によれば、無料版はファイアウォールのルールとマルウェアの検出情報を、有料版の利用者に提供してから30日後に受け取ります。新しい脆弱性が公開された直後の期間、無料版はその攻撃を知らない状態で動くということです。

無料版と有料版の違いを整理すると、次のようになります。製品によって細部は異なるため、導入前に公式情報を確認してください。

| 観点 | 無料版 | 有料版 |
|---|---|---|
| 基本機能（WAF・スキャン・二要素認証） | 使える | 使える |
| 脅威情報の更新頻度 | 遅れがある（Wordfenceは30日） | 随時 |
| サポート | フォーラム中心 | 問い合わせ窓口あり |
| 侵害時の復旧支援 | なし | 製品により提供 |

差が出るのは、公開直後の脆弱性が狙われる期間です。逆に言えば、更新を早く適用し、不要なプラグインを減らしておけば、この差の影響は小さくなります。無料版で足りるかどうかは、機能表ではなく、サイトが扱うデータと運用体制で判断してください。

## 8製品を役割で整理する

ここまで挙げた製品を、役割ごとに一覧にします。同じ列に並ぶ製品は機能が重なるため、そのなかから1つを選びます。

無料版で選べる主な製品は次のとおりです。掲載している機能は各製品の公式情報に基づきますが、改版によって変わるため、導入前に最新の記載を確認してください。

| 製品 | 役割 | 主な無料機能 | 日本語 |
|---|---|---|---|
| SiteGuard WP Plugin | 防御（ログイン保護） | ログインページURL変更、画像認証、ログインロック、管理ページアクセス制限、XMLRPC防御、更新通知 | 対応 |
| Limit Login Attempts Reloaded | 防御（ログイン保護） | ログイン試行回数の制限 | 一部 |
| Wordfence Security | 防御（総合） | ファイアウォール、マルウェアスキャン、二要素認証、ログイン保護 | 非対応 |
| All-In-One Security（AIOS） | 防御（総合） | ログイン保護、設定の強化ほか（範囲は公式情報を確認） | 一部 |
| Solid Security | 防御（総合） | ログイン保護、設定の強化ほか（範囲は公式情報を確認） | 一部 |
| UpdraftPlus / BackWPup | バックアップ | ファイルとデータベースの保存、外部ストレージ保存 | 一部 |
| Akismet Anti-Spam | スパム対策 | コメント・フォームのスパム判別（個人の非商用利用は無料） | 対応 |
| CyberNote Security Checker | 診断 | 10項目の診断（バージョン鮮度3・ハードニング7）、3段階表示 | 対応 |

防御の列には5つの製品が並びますが、選ぶのは1つだけです。ここから2つ以上を有効化すると、ログイン保護やWAFが二重に働き、管理者自身が締め出される可能性があります。

バックアップ・スパム対策・診断の3つは、防御系と処理が重ならないため、それぞれ1つずつ追加して構いません。

## ログイン保護に絞った無料プラグイン

ログイン保護は、費用をかけずに最も効果が出る領域です。総当たり攻撃やパスワードリスト攻撃は自動化されており、回数制限と画像認証を入れるだけで大半が成立しなくなります。

### SiteGuard WP Plugin

国産のプラグインで、管理画面もエラーメッセージも日本語です。開発元の公式情報によれば、ログインページのURL変更、画像認証、ログインロック（一定回数の失敗でIPアドレスを一時遮断）、ログイン詳細エラーメッセージの無効化、管理ページアクセス制限、XMLRPC防御、更新通知といった機能を備えています。

注意点が2つあります。1つは、有効化した時点でログインページのURLが自動的に変更されることです。新しいURLは管理者宛のメールでも通知されるため、必ず確認してブックマークしてください。もう1つは、WAFチューニングサポート機能が、開発元のサーバー向けWAF製品が導入されている環境を前提としている点です。この機能があること自体は、プラグインがWAFを提供していることを意味しません。

### Limit Login Attempts Reloaded

ログイン試行回数の制限に機能を絞った軽量なプラグインです。すでに他の対策を入れており、回数制限だけを足したい場合に向きます。機能が少ないぶん、他のプラグインとの競合も起きにくくなります。

ログイン保護系は、複数入れると競合します。ここで挙げた2つを同時に有効化すると、回数制限が二重にかかり、自分自身が遮断される可能性があります。どちらか1つを選んでください。

## 総合型の無料プラグイン

総合型は、WAF・マルウェアスキャン・ログイン保護を1つで担う製品です。導入は1つで済みますが、動作は重くなる傾向があります。表示速度を重視するサイトでは、導入前後で読み込み時間を比較してください。

### Wordfence Security

WordPress.orgの掲載情報によれば、無料版でもファイアウォール、マルウェアスキャナー、ログイン保護、二要素認証を利用できます。マルウェアスキャナーは、コアファイル・テーマ・プラグインを公式リポジトリの内容と比較し、変更されたファイルを報告します。

前述のとおり、無料版はファイアウォールのルールとマルウェア検出情報を30日遅れで受け取ります。また、国別のアクセス遮断やリアルタイムのIP遮断リストは有料版の機能です。攻撃のログを詳しく見たい場合には有力な選択肢ですが、この30日という遅延を理解したうえで使ってください。

### All-In-One Security（AIOS）／Solid Security

いずれも、ログイン保護やファイアウォール、設定の強化を無料版で提供する総合型のプラグインです。ただし、無料版に含まれる機能の範囲は改版によって変わるため、導入前にWordPress.orgの掲載ページで最新の記載を確認してください。

総合型を選ぶ際は、機能の多さではなく、自分が実際に使う機能で判断してください。使わない機能を有効化すると、正常なアクセスまで遮断してしまうことがあります。設定は段階的に有効化し、そのつどサイトの表示と管理画面を確認しながら進めます。

## バックアップとスパム対策の無料プラグイン

防御を固めても、侵害の可能性はゼロになりません。そのとき確実に戻せるかどうかが、被害の大きさを決めます。バックアップは、無料で実施できる対策のなかで最も効果が大きい部類です。

### UpdraftPlus／BackWPup

どちらも、ファイルとデータベースの両方をバックアップでき、外部ストレージへの保存に対応しています。重要なのは「取得しているか」ではなく「復元できるか」です。保存先をサーバー内だけにすると、サーバーごと侵害された際にバックアップも失われます。外部に保存し、四半期に一度は実際に復元を試してください。

### Akismet Anti-Spam

コメントやフォームへのスパム投稿を判別するプラグインです。個人の非商用利用であれば無料のプランを選べます。商用サイトでの利用条件は変更されることがあるため、公式の案内を確認してください。

この2つの役割は、防御系プラグインと機能が重なりません。同時に導入しても競合しないため、防御系を1つ選んだうえで追加してください。

## 設定の不備を可視化する無料プラグイン

防御系プラグインを入れていても、設定の不備は自動的には解消されません。既定の管理者名が残っている、デバッグ表示が有効のまま、管理画面からのファイル編集が有効になっている。こうした項目は、攻撃を遮断する機能とは別に、自分で確認して塞ぐ必要があります。

<!-- CTA -->
筆者が開発・公開しているCyberNote Security Checkerは、この確認に特化した無料の診断プラグインです。バージョン鮮度に関する3項目（WordPress本体、PHPバージョン、プラグイン・テーマの更新状況）と、ハードニング設定に関する7項目（デバッグ表示、ファイル編集機能、管理者ユーザー名、HTTPS、データベース接頭辞、XML-RPC、REST APIによるユーザー名列挙）の計10項目を診断し、要対応・改善推奨・問題なしの3段階で表示します。外部への通信を行わず、設定を自動変更しない読み取り専用の設計です。

このプラグインは診断ツールであり、攻撃を遮断するものではありません。すべての脆弱性を検出できるわけでもないため、防御用のプラグインやバックアップと組み合わせて使ってください。導入は、管理画面の「プラグイン」→「新規プラグインを追加」で `CyberNote` と検索します。

## 無料版を選ぶときの4つの判断軸

無料の製品は数が多く、機能表を並べても決まりません。次の4つの軸で絞ると、候補は2つか3つになります。

判断軸は、機能の多さではありません。導入後に運用を続けられるかどうかを決める要素です。

| 判断軸 | 見るポイント |
|---|---|
| 脅威情報の更新頻度 | 無料版で情報の受け取りに遅れがあるか。遅れの期間は公表されているか |
| サポート | 質問できる窓口があるか。無料版はフォーラム対応が中心になる |
| サーバー負荷 | 常時スキャンや常駐処理があるか。表示速度への影響を導入前後で比較する |
| 日本語対応 | 設定画面とエラーメッセージが読めるか。誤設定の事故は言語の壁から起きやすい |

とくに軽視されやすいのが4番目です。セキュリティプラグインは、設定を誤ると自分自身を締め出します。英語の設定画面を雰囲気で操作するくらいなら、機能が少なくても日本語の製品を選ぶほうが、結果として安全に運用できます。

サーバー負荷も確認してください。総合型の製品は機能が多いぶん、処理が重くなる傾向があります。導入前後で表示速度を測り、明確に遅くなるようであれば、機能を絞った製品への変更を検討します。

## 無料版で運用するときに必ず続けること

無料版の弱点は、新しい脅威への対応が遅れることでした。この遅れを埋めるのは、有料版への切り替えではなく、運用の習慣です。

やることは3つだけです。いずれも短時間で終わり、無料版の限界を実質的に補います。

- **週に1回、更新を確認する**：管理画面の「更新」を開き、保留中の項目を適用する。脅威情報の遅れがあっても、脆弱性そのものが修正されていれば影響を受けません
- **使っていないプラグイン・テーマを削除する**：停止だけでは、ファイルがサーバー上に残ります。プラグインの数が減れば、狙われる箇所も減ります
- **月に1回、設定を診断する**：新しいプラグインを追加した際に、設定の前提が崩れていないかを確認します

この3つを続けているサイトと、有料版を入れただけで放置しているサイトを比べれば、前者のほうが安全です。プラグインは対策の一部であって、対策そのものではありません。

## 無料だけで組む構成

役割ごとに1つずつ選べば、無料のプラグインだけで基本的な防御は成立します。防御系だけを増やすのではなく、守備範囲の違う4つを揃えるのが要点です。

以下は、個人ブログから小規模な企業サイトまでを想定した構成です。プラグインの総数は少ないほど安全であるため、この4つ以外にセキュリティ目的の追加は基本的に不要です。

| 役割 | 選択肢 |
|---|---|
| 防御（1つだけ） | SiteGuard WP Plugin または Wordfence Security |
| バックアップ | UpdraftPlus または BackWPup |
| スパム対策 | Akismet Anti-Spam |
| 診断 | CyberNote Security Checker |

日本語での設定に不安がある場合はSiteGuard、攻撃のログやスキャン結果を詳しく確認したい場合はWordfenceを選ぶ、という分け方が現実的です。どちらを選んでも、防御系は1つに絞ってください。

## 有料版が必要になる条件

無料版で足りなくなるのは、機能が不足したときではなく、脅威情報の遅れが許容できなくなったときです。判断の基準は、侵害されたときに何を失うかにあります。

次のいずれかに当てはまる場合、有料版や外部のサービスを検討する段階にあります。当てはまらないのであれば、無料の構成を正しく運用するほうが費用対効果は高くなります。

- 決済情報や会員の個人情報を扱っている（侵害時に報告義務が生じる可能性がある）
- サイトの停止が売上に直結する
- 複数のサイトを管理しており、手作業での確認が現実的でない
- 侵害時に自力で復旧する時間や技術が確保できない

該当しない個人サイトであれば、無料の4点構成に、週1回の更新確認と月1回の診断を組み合わせるだけで、実務上必要な水準には届きます。

## よくある質問

**Q. 無料のセキュリティプラグインだけで大丈夫ですか。**
A. 個人サイトや小規模サイトであれば成立します。ただし、更新の適用とバックアップを続けることが前提です。

**Q. 無料版と有料版の一番の違いは何ですか。**
A. 脅威情報の更新頻度です。Wordfenceの場合、無料版はファイアウォールのルールとマルウェア検出情報を30日遅れで受け取ります。

**Q. 無料の防御プラグインを2つ入れてもいいですか。**
A. 推奨しません。ログイン保護やWAFが競合し、自分自身がログインできなくなる可能性があります。

**Q. どの無料プラグインから入れるべきですか。**
A. 防御系を1つ、バックアップを1つ。この2つを先に入れてください。診断とスパム対策はそのあとで構いません。

## まとめ

WordPressセキュリティプラグインの無料版でも、WAF・マルウェアスキャン・二要素検証を含む基本的な防御は構築できます。有料版との差は機能の有無ではなく、脅威情報の更新頻度とサポートの範囲にあります。無料版を使う場合は、この遅れを前提に、更新の適用と不要プラグインの削除を早めに行ってください。構成は、防御・バックアップ・スパム対策・診断を1つずつ。防御系を2つ入れると競合します。
